# 🚀 What is Prometheus?

Prometheus is an open-source monitoring and alerting toolkit designed for reliability and scalability, especially in cloud-native environments like Kubernetes.

---

# 🧠 Prometheus Server

The **Prometheus Server** is the core component responsible for:

### 🔹 Data Scraping
- Periodically collects metrics from configured targets
- Targets expose metrics via HTTP endpoints

### 🔹 Data Storage
- Stores metrics in a **time-series database**
- Optimized for efficient storage and retrieval

### 🔹 Querying
- Uses **PromQL (Prometheus Query Language)**
- Supports aggregation, filtering, and calculations

---

# ⚙️ Prometheus Components

## 1. Prometheus Server
- Scrapes metrics
- Stores data
- Provides query interface

## 2. Pushgateway
- Used for **short-lived jobs**
- Allows pushing metrics instead of scraping
- Useful for batch jobs

## 3. Exporters

Exporters act as **translators/adapters** between third-party systems and Prometheus.

### ❗ Problem
Prometheus can only collect metrics from systems that expose data in its required format.

### ✅ Solution
Exporters convert metrics into Prometheus-compatible format.

### 🔄 How Exporters Work

1. Connect to third-party system (e.g., database, OS, web server)
2. Collect internal metrics
3. Convert metrics into Prometheus format
4. Expose metrics via an HTTP endpoint (e.g., `/metrics`)
5. Prometheus scrapes the endpoint

---

## 🖥️ Node Exporter Example

- **Third-party system:** Linux/Windows OS  
- **What it does:**
  - Reads CPU usage
  - Memory usage
  - Disk space
  - Network traffic  

- Uses system files (e.g., `/proc` in Linux)

---

## 📦 Common Exporters

| System        | Exporter            | Metrics Exposed |
|--------------|--------------------|----------------|
| PostgreSQL   | Postgres Exporter  | Queries, connections, locks |
| MySQL        | MySQL Exporter     | Buffer pool, query time |
| Nginx        | Nginx Exporter     | Request rate, connections |
| Redis        | Redis Exporter     | Memory usage, key stats |
| AWS CloudWatch | CloudWatch Exporter | EC2, RDS, S3 metrics |

---

# 🚨 Alertmanager

- Handles alerts generated by Prometheus
- Features:
  - Deduplication
  - Grouping
  - Routing

- Sends alerts to:
  - Email
  - Slack
  - PagerDuty
  - Other notification systems

---

# 🔍 Service Discovery

Prometheus supports automatic discovery of targets through:

- Static configuration
- DNS-based discovery
- Kubernetes integration
- Cloud provider integrations

---

# 📐 PromQL (Prometheus Query Language)

- Used to query time-series data
- Supports:
  - Arithmetic operations
  - Aggregations
  - Filtering

---

# 🔄 Data Flow in Prometheus

## 1. Scraping Metrics
Prometheus pulls metrics from targets at regular intervals.

## 2. Storing Metrics
- Stored as time-series data
- Identified by:
  - Metric name
  - Labels (key-value pairs)

## 3. Querying Metrics
- Using PromQL
- Accessed via:
  - Prometheus UI
  - HTTP API
  - Grafana dashboards

## 4. Alerting
- Rules evaluate metrics
- Alerts triggered when conditions are met

## 5. Alert Handling
- Alerts sent to Alertmanager
- Processed and routed to receivers

---

# 🔁 Prometheus Workflow Example

1. **Service Discovery**
   - Discovers pods, services, and nodes (in Kubernetes)

2. **Scraping**
   - Collects metrics from endpoints

3. **Storing**
   - Saves metrics in time-series database

4. **Querying**
   - Analyze data using PromQL

5. **Alerting**
   - Trigger alerts based on defined rules

6. **Alertmanager**
   - Deduplicates, groups, and sends notifications

---

# 🧾 Summary

- Prometheus uses a **pull-based monitoring model**
- Exporters act as **bridges between systems and Prometheus**
- PromQL enables powerful querying and analysis
- Alertmanager ensures efficient alert handling
- Ideal for **Kubernetes and cloud-native observability**
